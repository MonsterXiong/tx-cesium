## 📋 部署步骤

### 1. 准备部署包

```bash
# 构建项目
npm run build

# 打包（可选）
tar -czf react-template-dist.tar.gz dist/

# 或使用 zip
zip -r react-template-dist.zip dist/
```

### 2. 上传到服务器

```bash
# 方法 1: scp
scp -r dist/* user@server:/path/to/deploy/

# 方法 2: rsync
rsync -av --delete dist/ user@server:/path/to/deploy/

# 方法 3: 解压部署包
tar -xzf react-template-dist.tar.gz -C /path/to/deploy/
```

### 3. 配置 Web 服务器

**Nginx 配置示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/deploy;
    index index.html;
    
    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # Cesium 资源缓存（1年）
    location /cesium/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 静态资源缓存（1年）
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;
}
```

### 4. 重启服务

```bash
# Nginx
sudo systemctl restart nginx

# Apache
sudo systemctl restart apache2
```

1. **Gzip 压缩**（配置见上文）

   - 二次访问无需重新下载
   - 显著提升加载速度

## ⚠️ 常见问题处理

### 问题 1：地图显示黑屏

**可能原因**：
- Cesium 资源未正确部署
- 服务器配置问题

**排查方法**：
```bash
# 检查 Cesium 目录
ls /path/to/deploy/cesium/

# 检查资源访问
curl http://your-domain.com/cesium/Assets/approximateTerrainHeights.json
```

**解决方案**：
- 确保 `dist/cesium/` 完整部署
- 检查服务器配置

### 问题 2：Workers 加载失败

**错误信息**：
```
Failed to load worker script
```

**解决方案**：
```nginx
# Nginx 添加 MIME 类型
location /cesium/Workers/ {
    types {
        application/javascript js;
    }
}
```

### 问题 3：404 错误

**可能原因**：
- 文件路径不正确
- 服务器配置问题

**解决方案**：
```nginx
# 确保 SPA 路由配置正确
location / {
    try_files $uri $uri/ /index.html;
}
```

### 问题 4：跨域问题

**如果使用单独的静态服务器**：

```nginx
# 在静态服务器添加 CORS 头
location /cesium/ {
    add_header Access-Control-Allow-Origin *;
}
```
