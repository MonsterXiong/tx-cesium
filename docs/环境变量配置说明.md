# 环境变量配置说明

## 📋 环境变量文件

项目已创建以下环境变量文件：

```
.env.development    # 开发环境配置
.env.production     # 生产环境配置
.env.example        # 环境变量示例（可提交到 git）
```

## 🔐 Cesium Token 配置

### 1. 获取 Cesium Access Token

1. 访问 [Cesium Ion](https://ion.cesium.com/)
2. 注册/登录账号
3. 进入 **Access Tokens** 页面
4. 点击 **Create Token** 创建新的访问令牌
5. 复制生成的 Token

### 2. 配置到项目

打开 `.env.development` 文件，将获取的 Token 填入：

```env
VITE_CESIUM_ACCESS_TOKEN=你复制的_token_粘贴到这里
```

### 3. 验证配置

启动开发服务器后，访问 `/cesium-view` 页面，如果能正常显示地图，说明配置成功。

## 📝 环境变量说明

### 应用配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `VITE_APP_TITLE` | 应用标题 | React Template |
| `VITE_APP_BASE_API` | API 基础路径 | /api |
| `VITE_APP_PORT` | 开发服务器端口 | 3000 |

### Cesium 配置

| 变量名 | 说明 | 必需 |
|--------|------|------|
| `VITE_CESIUM_BASE_URL` | Cesium 资源路径 | 否 |
| `VITE_CESIUM_ACCESS_TOKEN` | Cesium Ion Token | 是* |

\* 虽然没有 Token 也能运行，但某些功能会受限

## 🚀 使用环境变量

### 在代码中使用

```typescript
// 获取环境变量
const apiUrl = import.meta.env.VITE_APP_BASE_API
const cesiumToken = import.meta.env.VITE_CESIUM_ACCESS_TOKEN

// TypeScript 类型支持已在 src/types/global.d.ts 中定义
```

### 类型定义

环境变量的 TypeScript 类型已在 `src/types/global.d.ts` 中定义：

```typescript
interface ImportMetaEnv {
  readonly VITE_APP_TITLE: string
  readonly VITE_APP_BASE_API: string
  readonly VITE_APP_PORT: number
  readonly VITE_CESIUM_BASE_URL: string
  readonly VITE_CESIUM_ACCESS_TOKEN: string
}
```

## 🔒 安全注意事项

### ⚠️ 不要提交敏感信息

`.env.development` 和 `.env.production` 已在 `.gitignore` 中被忽略，不会提交到 git 仓库。

### ✅ 可以提交的文件

- `.env.example` - 环境变量示例文件（不包含真实的敏感信息）

### ❌ 不能提交的文件

- `.env`
- `.env.local`
- `.env.development`
- `.env.development.local`
- `.env.production`
- `.env.production.local`

## 📚 环境变量优先级

Vite 的环境变量加载优先级（从高到低）：

1. `.env.[mode].local` - 本地环境特定配置（不会被提交）
2. `.env.[mode]` - 环境特定配置
3. `.env.local` - 本地配置（不会被提交）
4. `.env` - 所有环境共用配置

其中 `[mode]` 是：
- `development` - 开发环境（npm run dev）
- `production` - 生产环境（npm run build）

## 🎯 快速开始

### 第一次使用

1. 复制示例文件（如果需要）：
```bash
cp .env.example .env.development
```

2. 编辑 `.env.development`，填入你的配置

3. 启动开发服务器：
```bash
npm run dev
```

### 配置生产环境

编辑 `.env.production`，设置生产环境的配置：

```env
VITE_APP_BASE_API=https://your-production-api.com
VITE_CESIUM_ACCESS_TOKEN=your_production_token
```

## 💡 常见问题

### Q: Cesium 地图显示黑屏或报错？

A: 可能是 Token 配置问题，请检查：
1. Token 是否正确配置
2. Token 是否有效
3. 网络是否正常

### Q: 如何使用不同的 Token？

A: 创建 `.env.development.local` 文件，设置本地专用的 Token：

```env
VITE_CESIUM_ACCESS_TOKEN=your_local_token
```

### Q: 环境变量修改后不生效？

A: 需要重启开发服务器：
1. 停止当前服务器（Ctrl+C）
2. 重新运行 `npm run dev`

### Q: 如何在构建时使用不同的环境？

A: Vite 会根据命令自动选择：
- `npm run dev` → 使用 `.env.development`
- `npm run build` → 使用 `.env.production`
