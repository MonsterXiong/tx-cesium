# 内网部署说明

## 📋 项目配置

本项目已针对**内网环境**进行配置，Cesium 地图引擎完全使用本地资源，**不依赖任何外网 CDN**。

## ✅ 已完成的内网适配

### 1. Cesium 本地化配置

**开发环境**：
- Cesium 资源从 `node_modules/cesium` 加载
- 无需外网访问

**生产环境**：
- Cesium 资源会被自动打包到 `dist/cesium` 目录
- 部署后完全自包含，无需外网

### 2. 环境变量配置

```env
# .env.development 和 .env.production

# Cesium 配置 - 内网模式
VITE_CESIUM_BASE_URL=          # 留空，使用本地资源
VITE_CESIUM_ACCESS_TOKEN=       # 留空，内网环境通常不需要
```

### 3. 自动资源处理

通过 `vite-plugin-cesium` 插件：
- ✅ 自动复制 Cesium 静态资源
- ✅ 自动处理 Worker 文件
- ✅ 自动处理 WASM 文件
- ✅ 自动处理 Assets 资源

## 🚀 开发环境启动

### 1. 安装依赖

```bash
npm install
```

**注意**：
- Cesium 包较大（约 300MB），首次安装需要一些时间
- 内网环境需要配置 npm 私服或离线安装

### 2. 启动开发服务器

```bash

```

访问 http://localhost:3000

**验证 Cesium**：
- 访问 `/cesium-view` 页面
- 应该能看到 3D 地图正常显示
- 控制台显示：`[DEBUG_CESIUM] 内网模式：使用本地 Cesium 资源`

## 📦 生产环境构建

### 1. 构建项目

```bash
npm run build
```

### 2. 构建产物

```
dist/
├── index.html
├── assets/
│   ├── *.js        # JavaScript 文件
│   └── *.css       # CSS 文件
└── cesium/         # Cesium 资源（自动复制）
    ├── Assets/     # 地图数据、纹理等
    ├── Workers/    # Web Workers
    ├── ThirdParty/ # 第三方库
    └── Widgets/    # UI 组件
```

### 3. 部署注意事项

**Cesium 目录**：
- `dist/cesium/` 目录包含所有 Cesium 资源（约 200MB）
- 必须完整部署到服务器
- 保持目录结构不变

**服务器配置**：
```nginx
# Nginx 示例配置
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/dist;
    index index.html;
    
    # 处理 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # Cesium 资源缓存
    location /cesium/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # 静态资源缓存
    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

## 🔧 内网 npm 配置

### 方案 1：使用内网 npm 私服

```bash
# 配置私服地址
npm config set registry http://your-internal-npm-registry

# 安装依赖
npm install
```

### 方案 2：离线安装

```bash
# 在有网环境导出依赖
npm pack cesium
# 生成 cesium-x.x.x.tgz

# 在内网环境安装
npm install ./cesium-x.x.x.tgz
```

### 方案 3：使用 npm cache

```bash
# 在有网环境缓存依赖
npm cache add cesium

# 复制 ~/.npm 目录到内网环境
# 在内网环境安装（会从缓存读取）
npm install
```

## ⚡ 性能优化建议

### 1. 启用 Gzip 压缩

```nginx
# Nginx 配置
gzip on;
gzip_vary on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
gzip_min_length 1000;
```

**效果**：
- Cesium 资源可以压缩 60-70%
- 传输大小从 200MB 降至约 60-80MB

### 2. 启用浏览器缓存

```nginx
# Cesium 资源强缓存（版本更新时清除）
location /cesium/ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

### 3. 使用本地静态服务器

对于内网环境，可以将 Cesium 资源部署到专门的静态文件服务器：

```bash
# 文件服务器结构
static-server/
└── cesium/    # Cesium 公共资源
```

然后在应用中配置：
```env
VITE_CESIUM_BASE_URL=http://static-server/cesium
```

## 🐛 常见问题

### Q: Cesium 地图显示黑屏？

**排查步骤**：
1. 检查控制台是否有错误
2. 检查 `/cesium/` 资源是否能正常访问
3. 检查服务器配置是否正确处理静态资源

**解决方案**：
```bash
# 确认 Cesium 资源存在
ls dist/cesium/Assets

# 测试资源访问
curl http://localhost/cesium/Assets/approximateTerrainHeights.json
```

### Q: Workers 加载失败？

**原因**：服务器未正确配置 MIME 类型

**解决方案**：
```nginx
# Nginx 添加 MIME 类型
location /cesium/Workers/ {
    types {
        application/javascript js;
    }
}
```

### Q: 构建后 Cesium 资源缺失？

**原因**：`vite-plugin-cesium` 配置问题

**解决方案**：
```bash
# 清理后重新构建
rm -rf dist node_modules/.vite
npm run build
```

### Q: 内网安装依赖失败？

**解决方案**：
1. 配置内网 npm 私服
2. 使用离线依赖包
3. 复制 node_modules 目录

### Q: Cesium 版本升级？

```bash
# 升级到最新版本
npm update cesium

# 或指定版本
npm install cesium@1.110.0
```
