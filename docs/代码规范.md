## 📝 注释规范

### 1. 文件头部注释

```typescript
/**
 * 用户状态管理
 * 管理用户信息、登录状态等
 */
```

### 2. 函数/方法注释

**必须包含**：
- 功能描述
- 参数说明（如有）
- 返回值说明（如有）

```typescript
/**
 * 飞行到指定城市
 * @param city 城市名称
 * @returns void
 */
const flyToCity = (city: string): void => {
  // 实现代码...

 * 添加标记点
 * @param position 位置坐标 [经度, 纬度, 高度]
 * @param options 配置选项
 * @returns 实体对象
 */
export const addPoint = (
  position: [number, number, number],
  options?: PointOptions
): Cesium.Entity => {
  // 实现代码...
}
```

### 3. 接口/类型注释

```typescript
/**
 * 用户信息接口
 */
export interface UserInfo {
  /** 用户ID */
  id: string | number
  /** 用户名 */
  username: string
  /** 昵称 */
  nickname?: string
}

/**
 * 路由配置接口
 */
export interface RouteConfig {
  /** 路由路径 */
  path: string
  /** 路由元信息 */

```

### 4. 复杂逻辑注释

```typescript
// 处理重定向
if (redirect) {
  return <Navigate to={redirect} replace />
}

// 获取组件（懒加载）
const Component = routeComponents[path]
```

### 5. Console 调试注释

**统一使用前缀**便于筛选：

```typescript
console.log('[DEBUG_USER] 用户已登录:', userInfo.username)
console.log('[DEBUG_CESIUM] Viewer 初始化成功')
console.log('[DEBUG_THEME] 主题模式已切换:', mode)
console.error('[DEBUG_REQUEST] 请求失败:', error)
```

**前缀规范**：
- `[DEBUG_USER]` - 用户相关
- `[DEBUG_CESIUM]` - Cesium 相关
- `[DEBUG_THEME]` - 主题相关
- `[DEBUG_REQUEST]` - 请求相关
- `[DEBUG_STORAGE]` - 存储相关
- `[DEBUG_APP]` - 应用相关

## 🎨 CSS 样式规范

### 1. 优先使用 UnoCSS 工具类

```tsx
// ✅ 推荐：使用工具类
<div className="flex items-center justify-between p-4 bg-bg-container rounded-lg">
  内容
</div>

// ❌ 不推荐：行内样式
<div style={{ display: 'flex', padding: '16px' }}>
  内容
</div>
```

### 2. 使用 CSS Variables

```tsx
// ✅ 推荐：使用 CSS 变量
<div className="text-primary bg-bg-container border-border">
  内容
</div>

// ❌ 不推荐：硬编码颜色
<div className="text-blue-500 bg-white">
  内容
</div>
```

### 3. 使用预定义快捷方式

```tsx
// ✅ 推荐：使用快捷方式
<div className="flex-center">居中内容</div>
<div className="flex-between">两端对齐</div>
<div className="card-base">卡片样式</div>

// ❌ 不推荐：重复写完整类名
<div className="flex items-center justify-center">居中内容</div>
```

**项目预定义快捷方式**：
- `flex-center` - flex 居中
- `flex-between` - flex 两端对齐
- `flex-col-center` - flex 纵向居中
- `transition-base` - 基础过渡动画
- `card-base` - 卡片样式

### 4. 复杂样式使用独立 CSS 文件

```tsx
// ✅ 推荐：复杂样式独立
// CesiumViewer.tsx
import './cesium-viewer.css'

// cesium-viewer.css
.cesium-viewer-container {
  width: 100%;
  height: 100%;
  position: relative;
}

// ❌ 不推荐：在组件中写大量样式
<div style={{ width: '100%', height: '100%', position: 'relative' }}>
```

### 5. 响应式设计

```tsx
// ✅ 推荐：使用响应式前缀
<div className="w-full md:w-1/2 lg:w-1/3">
  内容
</div>

// 响应式断点：
// sm: 640px
// md: 768px
// lg: 1024px
// xl: 1280px
```

## 📦 静态资源引入规范

### 1. 图片资源

**小图标/SVG**：放在 `src/assets/`

```tsx
// ✅ 推荐：使用 import
import logo from '@/assets/logo.svg'

function App() {
  return <img src={logo} alt="Logo" />
}
```

**大图片/公共资源**：放在 `public/`

```tsx
// ✅ 推荐：直接引用 public 路径
<img src="/images/banner.jpg" alt="Banner" />
```

### 2. Antd 图标

```tsx
// ✅ 推荐：按需导入
import { UserOutlined, HomeOutlined } from '@ant-design/icons'

function Menu() {
  return (
    <>
      <HomeOutlined />
      <UserOutlined />
    </>
  )
}

// ❌ 不推荐：全量导入
import * as Icons from '@ant-design/icons'
```

### 3. 字体文件

**放在 `public/fonts/`**

```css
/* styles/global.css */
@font-face {
  font-family: 'CustomFont';
  src: url('/fonts/custom-font.woff2') format('woff2');
}
```

### 4. Cesium 资源

**由 vite-plugin-cesium 自动处理**，无需手动引入

```typescript
// ✅ 推荐：直接使用
import * as Cesium from 'cesium'

const viewer = new Cesium.Viewer('cesiumContainer')
```

### 5. 路径别名

**使用 `@` 别名**：

```typescript
// ✅ 推荐：使用别名
import { useTheme } from '@/hooks'
import { storage } from '@/utils/storage'
import type { UserInfo } from '@/types/global'

// ❌ 不推荐：相对路径
import { useTheme } from '../../../hooks'
```

## 🔤 命名规范

### 1. 文件命名

#### 1.1 React 组件文件
**使用 PascalCase（大驼峰）**

```
✅ 正确示例：
   - CesiumViewer.tsx        # 组件文件
   - BasicLayout.tsx         # 布局组件
   - UserCard.tsx            # 业务组件
   - ProtectedRoute.tsx     # 路由组件

❌ 错误示例：
   - cesiumViewer.tsx        # 小驼峰
   - basic-layout.tsx        # 短横线
   - user_card.tsx          # 下划线
```

#### 1.2 页面文件
**文件夹使用 kebab-case，主文件使用 index.tsx**

```
✅ 正确示例：
   pages/
   ├── cesium-view/         # 文件夹：kebab-case
   │   └── index.tsx        # 主文件：index.tsx
   ├── dashboard/
   │   └── index.tsx
   └── not-found/
       └── index.tsx

❌ 错误示例：
   pages/
   ├── CesiumView/          # 文件夹不应使用 PascalCase
   └── cesiumView/          # 文件夹不应使用 camelCase
```

#### 1.3 工具函数和 Hooks
**使用 camelCase（小驼峰）**

```
✅ 正确示例：
   - useTheme.ts            # Hook 文件
   - useCesium.ts           # Hook 文件
   - storage.ts             # 工具函数
   - helpers.ts             # 工具函数
   - request.ts             # 请求工具

❌ 错误示例：
   - UseTheme.ts            # 不应使用 PascalCase
   - use-theme.ts           # 不应使用 kebab-case
```

#### 1.4 样式文件
**使用 kebab-case（短横线）**

```
✅ 正确示例：
   - cesium-viewer.css      # 组件样式
   - global.css             # 全局样式
   - reset.css              # 重置样式
   - variables.css          # 变量文件

❌ 错误示例：
   - cesiumViewer.css       # 不应使用 camelCase
   - CesiumViewer.css       # 不应使用 PascalCase
```

#### 1.5 配置文件
**使用 camelCase 或 kebab-case**

```
✅ 正确示例：
   - config.ts              # 配置文件
   - vite.config.ts         # 构建配置
   - uno.config.ts          # UnoCSS 配置
   - tsconfig.json          # TypeScript 配置

❌ 错误示例：
   - Config.ts              # 不应使用 PascalCase
```

#### 1.6 类型定义文件
**使用 kebab-case.d.ts**

```
✅ 正确示例：
   - api.d.ts               # API 类型定义
   - global.d.ts            # 全局类型定义
   - vite-env.d.ts          # Vite 环境类型

❌ 错误示例：
   - api.ts                 # 类型定义应使用 .d.ts
   - Api.d.ts               # 不应使用 PascalCase
```

#### 1.7 目录结构中的文件组织

```
components/
├── common/                 # 通用组件目录
│   ├── Button/
│   │   ├── index.tsx      # 组件主文件（PascalCase 组件名）
│   │   └── style.css      # 组件样式（kebab-case）

│   └── components/        # 页面专用组件
│       └── Banner.tsx     # 组件使用 PascalCase
```

#### 1.8 命名规范总结表

| 文件类型 | 命名规范 | 示例 |
|---------|---------|------|

## 🚀 实战示例

### 完整的组件示例

```typescript
/**
 * 用户卡片组件
 * 展示用户基本信息
 */

import { Card, Avatar } from 'antd'
import { UserOutlined } from '@ant-design/icons'
import type { FC } from 'react'

/**
 * 用户卡片 Props
 */
interface UserCardProps {
  /** 用户名 */
  name: string
  /** 头像地址 */
  avatar?: string
  /** 点击事件 */
  onClick?: () => void
}

/**
 * 用户卡片组件
 * @param props 组件属性
 * @returns React 组件
 */
const UserCard: FC<UserCardProps> = ({ name, avatar, onClick }) => {
  /**
   * 处理卡片点击
   */
  const handleClick = () => {
    console.log('[DEBUG_USER] 点击用户卡片:', name)
    onClick?.()
  }

  return (
    <Card 
      className="card-base cursor-pointer hover:shadow-lg transition-base"
      onClick={handleClick}
    >
      <div className="flex-center flex-col gap-4">
        <Avatar size={64} icon={<UserOutlined />} src={avatar} />
        <span className="text-lg font-bold text-text">{name}</span>
      </div>
    </Card>
  )
}

export default UserCard
```

### 完整的工具函数示例

```typescript
/**
 * 日期格式化工具函数
 */

/**
 * 格式化日期时间
 * @param date 日期对象或时间戳
 * @param format 格式化模板，默认 'YYYY-MM-DD HH:mm:ss'
 * @returns 格式化后的字符串
 */
export const formatDate = (
  date: Date | number | string,
  format = 'YYYY-MM-DD HH:mm:ss',
): string => {
  const d = new Date(date)

  // 提取日期组件
  const year = d.getFullYear()
  const month = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  const hour = String(d.getHours()).padStart(2, '0')
  const minute = String(d.getMinutes()).padStart(2, '0')
  const second = String(d.getSeconds()).padStart(2, '0')

  // 替换格式化模板
  return format
    .replace('YYYY', String(year))
    .replace('MM', month)
    .replace('DD', day)
    .replace('HH', hour)
    .replace('mm', minute)
    .replace('ss', second)
}
```
