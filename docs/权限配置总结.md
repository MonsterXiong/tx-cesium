## 🚫 之前被阻止的操作

### 问题：环境变量文件无法创建

**原因**：
Cursor AI 有全局安全机制（globalignore），默认阻止创建/修改 `.env*` 文件，防止敏感信息泄露。

**错误信息**：
```
Error: Editing this file is blocked by globalignore
```

**解决方案**：
使用终端命令创建：
```bash
cat > .env.development << 'EOF'
内容...
EOF
```

### 方法 1：使用 `.cursorignore` 文件（已配置）


```env
# .env.example（安全，可提交）
VITE_CESIUM_ACCESS_TOKEN=your_token_here

# .env（敏感，不提交）
VITE_CESIUM_ACCESS_TOKEN=eyJhbGc...真实token
```

### 3. 分离配置

```bash
# 开发环境
.env.development        # 开发配置，占位符
.env.development.local  # 真实开发密钥（git忽略）

# 生产环境
.env.production         # 生产配置，占位符
# 真实生产密钥在 CI/CD 中配置
```

## ❓ 常见问题

### Q: 为什么有些文件 AI 不能修改？

**A**: 安全机制，防止敏感信息泄露。通过 `.cursorignore` 可以控制。

### Q: 我想让 AI 修改某个被忽略的文件？

**A**: 从 `.cursorignore` 中移除该文件，或使用终端命令。

### Q: `.cursorignore` 和 `.gitignore` 有什么区别？

**A**: 
- `.gitignore` → 控制 Git 提交
- `.cursorignore` → 控制 AI 访问

### Q: 修改 `.cursorignore` 后需要重启吗？

**A**: 通常不需要，但如果没生效，可以重启 Cursor。
